# Claude Skill Porter — гайд на русском

## Что это такое

`Claude Skill Porter` — это инструмент для переноса skills из Claude Code в формат, который можно использовать в Codex.

Он не обещает полную совместимость.

Его задача проще и честнее:

- перенести то, что переносится безопасно
- отдельно показать всё спорное
- не скрывать места, где нужна ручная доработка

## Когда он нужен

Используй его, если:

- в проекте есть `.claude/skills/...`
- есть старые `.claude/commands/*.md`
- ты хочешь переехать в Codex
- тебе нужен не “магический импорт”, а понятный отчёт о рисках

## Что он умеет

Сейчас инструмент уже умеет:

- переносить один Claude skill
- переносить один legacy command
- мигрировать весь repo в batch-режиме
- делать safe auto-rewrite типовых конструкций
- писать markdown- и JSON-отчёты
- падать в `--strict`, если остались ручные проблемы

## Что переносится хорошо

Обычно автоматически и безопасно переносятся:

- имя skill
- описание
- основной markdown из `SKILL.md`
- папки `scripts/`, `references/`, `assets/`
- простые `allowed-tools`
- простой `argument-hint`

Это “зелёная зона”.

## Где начинаются сложности

Проблемные места связаны не с текстом, а с поведением Claude Code.

Самые важные примеры:

- `context: fork`
- `hooks`
- `agent`
- `paths`
- `model`
- `effort`
- `$ARGUMENTS`
- `${CLAUDE_*}`
- shell-вставки через `!`

Такие штуки нельзя честно назвать “полностью перенесёнными”, потому что в Codex они могут работать иначе или не иметь прямого аналога.

## Что лежит на выходе

После миграции для каждого элемента создаются:

- `SKILL.md`
- `agents/openai.yaml`
- `references/migration-report.md`
- `references/migration-manifest.json`
- `references/original-frontmatter.md`

В batch-режиме дополнительно появляются:

- `batch-migration-report.md`
- `batch-migration-manifest.json`

## Зачем нужен migration report

`migration-report.md` нужен, чтобы человек сразу видел:

- что перенеслось автоматически
- что перенеслось частично
- что требует ручной доработки
- почему это риск

Это защита от ложного ощущения, что “раз папка создалась, значит всё уже работает”.

## Главные риски простыми словами

Самые важные риски такие:

- файлы скопировались, а поведение стало другим
- скрытая логика Claude пропала
- аргументы стали непонятными после переноса
- в `references/` или `scripts/` остались приватные данные
- кто-то решит, что проект обещает полную совместимость, хотя это не так

Именно поэтому у проекта есть `--strict`, manifests, тесты и отдельная документация по рискам.

## Что уже есть в проекте

Сейчас в репозитории уже есть:

- рабочий Codex skill
- конвертер `convert_claude_skill.py`
- fixture-based тесты
- golden outputs
- CI
- документация по рискам
- гайд по GitHub-публикации

То есть это уже не идея “на словах”, а нормальный узкий `v1`.
