# Публикация на GitHub

Это короткий и безопасный путь публикации проекта без `gh` CLI.

## Что уже готово локально

В проекте уже есть:

- git-репозиторий
- ветка `main`
- первый коммит
- `README.md`
- `LICENSE`
- `.gitignore`
- GitHub Actions workflow

## Что нужно сделать на сайте GitHub

1. Зайти на GitHub.
2. Нажать `New repository`.
3. Указать имя репозитория.
4. Не ставить галочки:
   - `Add a README file`
   - `Add .gitignore`
   - `Choose a license`
5. Создать пустой репозиторий.

Это важно, чтобы GitHub не создал лишний стартовый коммит поверх уже готового локального проекта.

## Что сделать после создания пустого репозитория

Скопируй URL нового репозитория и выполни:

```bash
git remote add origin <URL_НОВОГО_GITHUB_РЕПОЗИТОРИЯ>
git push -u origin main
```

## Что проверить после push

- открывается `README.md`
- во вкладке `Actions` стартовал CI
- репозиторий показывает ветку `main`
- файлы `skills/`, `tests/`, `.github/workflows/ci.yml` на месте

## Если GitHub попросит авторизацию

Это нормально. Обычно GitHub предложит:

- войти в браузере
- использовать personal access token вместо пароля

Если дойдём до этого шага вместе, лучше уже работать от конкретного URL репозитория.
